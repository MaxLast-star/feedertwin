# Этап 5. Интерактивный what-if дашборд: заметки

## Решение

Вместо серверного приложения (Streamlit) собран **автономный статический
HTML** (`dashboard/index.html`, ~40 КБ): метрики предрассчитаны сеткой
и встроены в файл как JSON, интерактив — ванильный JS + Plotly (CDN).
Преимущества для портфолио: хостится на GitHub Pages без инфраструктуры,
открывается двойным кликом, встраивается в Docusaurus (этап 6).

## Состав

Две вкладки:

1. **Контур контроля** — сетка 4 стратегии × 11 интенсивностей
   (3–8 дет/с, шаг 0,5) × 12 порогов (0,40–0,95, шаг 0,05), каждая ячейка —
   Monte Carlo 6 репликаций × 300 с (528 ячеек, 3168 прогонов).
   Показатели: дефекты/1000, производительность, duty заслонки,
   ложные срабатывания, латентность. Графики: дефекты от интенсивности
   (все стратегии, маркер выбранной точки) и компромисс порога.
2. **Вибротранспортирование** — сетка 3 амплитуды × 36 частот (10–45 Гц)
   × 16 углов (0–15°), тепловая карта v(α, f) с линиями порога Γ = 1
   и рабочей точки НИР; Γ и критическая частота считаются на лету
   в браузере по формуле Γ = A·ω²/g.

## Дизайн

Эстетика инженерного чертежа: миллиметровочная сетка фона, чернильно-синяя
палитра с сигнальным оранжевым, моноширинная типографика подписей
(IBM Plex Mono), подвал — основная надпись (штамп) с реквизитами проекта.

## Воспроизводимость

`python scripts/build_dashboard.py` (~5–8 минут) детерминированно
пересобирает файл: зёрна Monte Carlo фиксированы.
