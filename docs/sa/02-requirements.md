# Требования к модели

Требования сформулированы к вычислительной модели как к инженерному
артефакту: что она обязана уметь и каким критериям качества отвечать.

## Функциональные требования

| ID | Требование |
|---|---|
| FR-1 | Рассчитывать установившуюся скорость подачи детали как функцию амплитуды, частоты, угла наклона и коэффициента трения |
| FR-2 | Определять режим транспортирования (безотрывный / подбрасывание) по коэффициенту Γ = A·ω²/g |
| FR-3 | Имитировать стохастический поток деталей с регулируемой интенсивностью и долей «сдвоек» |
| FR-4 | Моделировать контур «камера → детектор → заслонка» с временны́ми задержками из НИР |
| FR-5 | Поддерживать сменные стратегии управления через единый интерфейс |
| FR-6 | Считать показатели: производительность, дефекты подачи, латентность коррекции, загрузку заслонки, ложные срабатывания |
| FR-7 | Оценивать показатели методом Монте-Карло с доверительными интервалами |
| FR-8 | Ранжировать альтернативы методами AHP (с контролем согласованности) и TOPSIS |
| FR-9 | Строить анализ чувствительности результата к весам критериев |
| FR-10 | Предоставлять интерактивное представление результатов (what-if дашборд) |

## Нефункциональные требования

| ID | Требование | Проверка |
|---|---|---|
| NFR-1 (воспроизводимость) | Результаты детерминированы при фиксированном зерне ГСЧ | тест `test_reproducibility` |
| NFR-2 (верифицируемость) | Методы MCDA дают эталонные значения на примерах из литературы | тесты `test_ahp_saaty_reference_example`, `test_topsis_*` |
| NFR-3 (физическая корректность) | Деталь не проникает под лоток; нет отрицательных скоростей при подаче под уклон | тесты `test_particle_never_below_tray` и др. |
| NFR-4 (тестовое покрытие) | Каждый модуль покрыт модульными тестами; CI зелёный на каждом коммите | GitHub Actions |
| NFR-5 (переносимость) | Запуск на Python ≥ 3.11, Windows/Linux, без проприетарных зависимостей | матрица CI |
| NFR-6 (производительность) | Полная пересборка дашборда укладывается в ~8 минут на типовом ПК | замер `build_dashboard` |
| NFR-7 (сопровождаемость) | Код проходит статический анализ (ruff) без замечаний | шаг CI «Lint» |

## Допущения и ограничения

- деталь моделируется материальной точкой с кулоновским трением;
- качество детектора задаётся статистически (Beta-распределение
  уверенности), без привязки к конкретным весам нейросети;
- модель не валидирована натурным экспериментом — количественные значения
  следует трактовать как сравнительные оценки сценариев, качественные
  пороги (Γ = 1, tg α = μ) — как строгие следствия принятых допущений.
